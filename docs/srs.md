# 📑 Especificación de Requisitos (SRS)

## Historias de Usuario 
1. **HU-01: Control de Inventario:** Como administrador, quiero clasificar medicamentos por categorías para mantener un orden lógico según el tipo de fármaco.
2. **HU-02: Seguridad:** Como usuario, quiero un login estético y funcional que me permita acceder al panel administrativo de forma segura.
3. **HU-03: Facturación Eficiente:** Como cajero, quiero filtrar los medicamentos por categorías durante la venta para agilizar el servicio al cliente.

## Requisitos No Funcionales
* **Rendimiento:** El filtrado de categorías debe realizarse en tiempo real sin recargar la página.
* **Persistencia:** Los datos deben guardarse localmente mediante la API del navegador (LocalStorage).
* **Diseño:** Interfaz minimalista con paleta de colores corporativa (Slate/Blue).
